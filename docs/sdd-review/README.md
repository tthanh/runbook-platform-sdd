# SDD method review

A look at how this project uses Spec-Driven Development (SDD): what works, what
slipped, and what to fix. Written 2026-06-28.

This folder is notes *about* the method. It is not part of the Spec Kit pipeline
(specs/ and src/) and not cross-feature truth (docs/architecture.md). It sits in
docs/ as working territory, like the rest of docs/.

## Files

- [findings.md](findings.md) — the review in plain English: what's good, what's
  wrong, what to improve.
- [cleanup-plan.md](cleanup-plan.md) — the fixes that came out of the review, with
  status.
- [proposed-constitution-amendment.md](proposed-constitution-amendment.md) — one
  change that touches the constitution. Drafted as a proposal for a human to
  accept, because the agent never changes constitution state on its own.
